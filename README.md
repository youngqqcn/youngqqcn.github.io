# 说明

借助github action的功能, 自动将编译出静态文件部署在`master`分支,省去了手动部署的麻烦

> https://hexo.io/docs/github-pages


如果修改或者增加了文章,直接提交到远程`source`分支即可, github会自动进行编译, 将静态文件放在`master`分支


# 关于图床

最好适用github作为图床，虽然慢一点，但是能够保证图片不会丢失。

picgo工具很方便。
